# review-pr: コードレビュースキル

指定された PR のコードレビューを行い、結果を GitHub にコメント投稿する。

## 引数

- `$ARGUMENTS`: PR 番号（例: `45`）

## 手順

1. **差分取得**: `gh pr diff $ARGUMENTS` で差分を取得する
2. **PR 情報取得**: `gh pr view $ARGUMENTS --json title,body,author` で概要を把握する
3. **関連ファイルを必要に応じて読む**: 差分だけでは文脈が不足する場合は `Read` / `Grep` で補足する
4. **以下の観点で分析する**:
   - 🔴 要修正: バグ・論理的な誤り・セキュリティ問題
   - 🟡 検討事項: 設計・可読性・テストの網羅性
   - 🟢 良い点: 適切な実装
5. **コメント投稿**: 以下のコマンドで結果を投稿する（`gh-claude` スキルは使わず `gh` コマンドを直接使う）

```bash
gh pr comment $ARGUMENTS --body "..."
```

## 注意事項

- **`gh-claude` スキルは使用しない**。GitHub 操作には `gh` コマンドを直接 `Bash` ツールで実行すること
- 要修正がない場合も必ずコメントを投稿すること
- バイナリファイル（画像・アイコン等）の差分は内容ではなく構成・配置の観点でレビューする

## コメントフォーマット

```
## コードレビュー

### 🔴 要修正
（なければ「なし」と記載）

---

### 🟡 検討事項
（あれば箇条書き。なければセクションごと省略可）

---

### 🟢 良い点
（適切な実装を箇条書き）
```
